Техника минификации JS кода

Когда я был новичком в JS, чтобы посмотреть как сделать какую нибудь фичу я смотрел исходный код сайта и клиенсткий код, и таким образом учился о новых трюках JS. Но когда встречал минифицированный код то приходилось отклонять идею подсмотреть код, т.к. тогда еще не мог читать такой код, слишком сложно было.

Недавно для одного проекта стал портатировать лайкли социокнопки на ванильный JS, т.к. мне не хотелось тащить jQuery ради кнопок, но кнопки были отпадные. За это время, пока портатировал лайкли с jQuery на ванильный JS, я выучил технику минификации кода у минификатор minifier. О чем я вам и расскажу.

Внимание: данный пост не явялется призыв к действию и не рекомендует использовать показанную технику в своем коде. Данный пост написан с целью объяснить/показать как минификатор экономит место.

Сжатие выражений

Самое простое это сжать несколько выражений. Сжимая выражения, минифиактор может сократить код от ; и от фигурных скобок.

Чтобы сжать несколько выражений вроде вызовов функций/методов, присваивание значения к уже определенной переменной можно воспользоватся запятой.

Было 3 вызова к методам на каждой строчке.

this.getUserInput();
this.processUserInput();
this.doSomethingElse();

Стало тех же самых три вызова, только мы сэкономили один символ на последней ;.

this.getUserInput(),
this.processUserInput(),
this.doSomethingElse()

Выигрыш небольшой, но, зато если есть много методов можно сэкономить очень много места, к тому же эта техника работает хорошо с ниже описаными техниками.

Сжатие литералов

true, false и undefined тоже можно сжать. true и false сжимается посредством опретаором "не" !. Вот простая функция которая определяет является ли переменная булевом.

function is_bool (bool) {
    return bool === true || bool === false;
}

Чтобы сжать true и false достаточно воспользоватся оператором "не" и 0 или 1:

/**
 * !0 === true
 * !1 === false
 */
function is_bool (bool) {
    return bool === !0 || bool === !1;
}

А вот undefined можно сжать двумя способами: если есть замыкание через аргумент, через свойство объекта или же через выражение void 0.
